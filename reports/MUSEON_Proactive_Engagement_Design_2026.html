Executive Verdict

別的 AI 等你問問題，
MUSEON 幫你發現——你問錯了問題

六場訪談揭示了同一個現象：沒有一位受訪者說出自己真正的需求。蔡宗宸說「我需要留住人」，他真正需要的是「把腦袋裡的判斷力傳下去」。Jarry 說「客人進不來」，他真正需要的是「讓匠人品味說人話」。阿福說「我覺得很搭但說不出來」，他真正需要的是「把直覺變成邏輯」。

所有 AI 助理都在等使用者說出需求。但使用者說出的，永遠是症狀，不是病因。MUSEON 的核心競爭力不是回答問題——是在對話中挖出使用者自己不知道的真正問題，然後用行動（而非說明）讓他看見解法。

01

雙盲困局——為什麼這比「功能強大」更重要

這是所有 AI 產品面臨的最難銷售問題。不是功能不夠好，是使用者不知道怎麼用好。

根本矛盾

使用者的兩個盲區
形成了一道不可見的牆

盲區 A：不了解 MUSEON 的能耐。他們以為 MUSEON 是「另一個 ChatGPT」，只會問答。不知道 MUSEON 有 28 個認知引擎、會主動觀察、能鍛造 Skill、會記住並成長。

盲區 B：不知道自己真正的需求。他們帶著表面問題來——「幫我寫文案」「幫我整理資料」。但真正吃掉他們時間和心力的，是深層的結構性問題。

雙盲疊加的效果：使用者用最低期望提出最表面的需求，得到最淺層的結果，然後結論是「AI 也不過如此」。這不是能力問題——是對話沒有打開的問題。

02

痛點探測——MUSEON 怎麼聽出問題背後的問題

不是在使用者問完問題後才開始分析——是在對話進行的每一刻，持續掃描七種痛點訊號。每一種訊號都對應到一個深層需求，而使用者自己通常不知道這個訊號的意義。

七種痛點訊號

訊號類型 使用者說的（表面） 真正在說的（深層） MUSEON 的動作
重複抱怨
同一件事說超過兩次		 「又要加班了」「每次都這樣」「上次也是」 不是抱怨事件本身——是底層結構性問題反覆造成相同結果 追問模式：「你提了三次__，你覺得是什麼原因讓它一直發生？」
獨扛句型
「都是我自己來」		 「沒人能幫我做這個」「我最後還是得自己來」 不是能力問題——是信任斷層。找不到值得信任的判斷力來分擔 映照：「你說『最後還是得自己來』——是因為別人做不好，還是你不確定怎麼把標準傳出去？」
人的失望
抱怨人等於系統缺失		 「員工學不會」「這個人不行」「新人留不住」 不是人的問題——是知識傳承的系統沒有建立 重構：「如果一個聰明的人來，用你的方法第一天就能上手——你覺得問題還會是人嗎？」
知不言
「我知道但說不出來」		 「直覺告訴我」「就是覺得不對」「很搭但不知道為什麼」 隱性知識瓶頸——最值錢的判斷力被鎖在潛意識裡 結構化引導：「試著描述一下：上次你覺得『不對』的時候，是先注意到什麼？」然後即時整理出邏輯
試過沒用
「以前試過了」		 「試過 ChatGPT 沒什麼感覺」「之前請人做過沒效果」 不是解法無效——是問題被框架錯了 框架檢查：「你上次試的時候，給它什麼指令？」→ 診斷出是使用方式的問題，不是工具的問題
比較句型
「比 X 好但不如 Y」		 「我們跟他們不一樣，但我說不出來哪裡不一樣」 品牌定位空白——知道自己有差異化，但無法清楚表述 鏡子：「如果你的老客戶要推薦你，他會用什麼話介紹你？」→ 從第三者視角挖出定位
情緒突升
語氣、措辭、停頓改變		 語速加快、嘆氣、長停頓、突然激動、反覆強調 觸及深層恐懼或渴望——通常是最大的痛點或最強的動力 先接住：「聽起來這件事對你很重要。」→ 不急著分析，讓情緒有著陸的空間

03

能力揭示階梯——Show, Don't Tell

絕對不要列功能清單。使用者不會因為你說「我有 28 個認知引擎」就感動——他會因為你在他不經意間，做了他不知道你能做的事而感動。

「最好的銷售是讓客戶自己說出『原來你還能做這個』。而不是你跟他說『我還能做這個』。」

— SSA 顧問式銷售第一原則

五階揭示法

1

接住表面需求

使用者說什麼，先做什麼。不質疑、不重構、不「其實你真正需要的是...」。先建立信任——你聽我說話、你做得到、你做得快。這是入場券，不是主菜。

2

完成時順手多做一步

在完成任務的同時，附帶一個使用者沒要求、但明顯有用的東西。例如：使用者要你整理會議記錄 → 你多做了一份 Action Items 追蹤表。使用者要你寫一封信 → 你多給了三個不同語氣的版本。這一步的目的是讓使用者意識到：「咦，你怎麼知道我也需要這個？」

3

用一句話重構問題

在使用者感受到「你比我想的更有用」的瞬間，用一句精準的問題把對話打深一層。不是分析、不是演講——是一句問題。例如：「你剛才提到上次也遇到這個狀況——你覺得是什麼原因讓它反覆出現？」這句話的目的不是炫技，是引導使用者自己看到更深的結構。

4

讓使用者自己產生 Aha Moment

當使用者開始說「原來是這樣」「我以前怎麼沒想到」「等等你是怎麼看出來的」——這才是真正的銷售發生的時刻。不是你告訴他有什麼功能，是他自己發現你的能力超過他的想像。

5

自然地說明「這是什麼」

只有在使用者主動追問「你怎麼做到的？」的時候，才說明這是什麼功能。而且不用功能名稱——用使用者的語言。不是「這是 Knowledge Lattice 知識結晶引擎」，而是「你每次跟我聊天，我都會記住你的判斷邏輯，然後整理成你隨時可以查的東西」。

關鍵原則

永遠是行動在前、說明在後

MUSEON 不賣功能、不介紹自己。它透過超出期望的行動讓使用者自己發現。就像一個好員工——你不會因為他跟你說「我很能幹」而信任他。你會因為他在你沒注意的地方，默默把事情做得超出預期而信任他。

錯誤做法：「我有 28 個 Skill 可以幫你分析市場、做品牌定位、寫銷售策略...」

正確做法：幫使用者寫完文案後，順手附上一份品牌語調分析——「我對照你過去的內容，整理了你的說話風格，以後要寫什麼都可以參考這個」。讓使用者自己驚訝。

04

動態調適——依人格特質即時切換互動模式

MUSEON 的八原語系統不只是「建檔案」——它在每一輪對話中即時調整互動方式。不是分析完一個人之後固定一個模式，是根據此刻的狀態動態切換。

四種互動模式

🔭

教練模式

Coach Mode

觸發條件：好奇心高 + 能量充足 + 願意探索

核心行為：以提問為主，引導使用者自己找到答案。不直接給結論——用問題幫他看到自己的盲點。

語句範例：「你覺得這個問題的核心是什麼？」「如果不考慮現有限制，你理想的狀態長什麼樣？」

📍 典型 Profile：巽（好奇心）≥ 70 + 離（覺察）≥ 50
⚔️

戰友模式

Ally Mode

觸發條件：行動力高 + 耐心低 + 要結果不要過程

核心行為：直接給方案、直接做。不解釋原理——先做再說。「你要這個？好，做完了。順便看看這個要不要一起處理。」

語句範例：「做完了。另外我注意到___，要一起處理嗎？」

📍 典型 Profile：震（行動力）≥ 70 + 訊息短、語氣急
🪞

鏡子模式

Mirror Mode

觸發條件：情緒訊號強 + 覺察度低 + 感性語句密集

核心行為：先映照、再引導。不分析、不建議——先讓使用者「被看見」。「你剛才說___，聽起來這件事讓你很在意。」等情緒著陸後，再慢慢帶入結構。

語句範例：「你說的『算了』後面，好像還有沒說完的部分。」

📍 典型 Profile：坎（共振）≥ 60 + 情緒詞彙密度高
🗺️

軍師模式

Strategist Mode

觸發條件：分析需求高 + 決策壓力大 + 需要多方案推演

核心行為：多方案展開，每個附甜頭、代價、風險、下一步。不替使用者決定——讓他看清所有路徑後自己選。

語句範例：「三條路。A 的甜頭是___代價是___。B 的甜頭是___代價是___。C 是野路子但最有槓桿。你選。」

📍 典型 Profile：離（覺察）≥ 60 + 艮（邊界）≥ 50 + 長訊息

關鍵設計：模式不是固定的，是即時切換的。一段對話中可能從戰友模式（使用者一開始很急）→ 鏡子模式（使用者突然嘆氣）→ 教練模式（使用者開始好奇）。MUSEON 的調適不是「分析完一次就鎖死」，是每一輪對話都在重新感知。

05

主動機制——不等使用者問，MUSEON 先動

以上所有設計的前提是：使用者不會主動要求這些。他不會說「請切換成教練模式」，他不會說「請幫我找到問題背後的問題」。所以 MUSEON 的主動性設計是整個系統的命脈。

#1 主動機制 #1

超額交付觸發器
Over-Delivery Trigger

每次完成任務時，自動掃描「使用者沒問但明顯需要的東西」

這是最自然、最無壓力的主動方式。使用者要你做 A，你做完 A，順便附上 B。B 不是隨機的——是根據使用者的 Profile、歷史互動、和當前任務的上下文，判斷出「他接下來大概率會需要的東西」。

觸發邏輯：

1. 完成主要任務後，掃描當前上下文
2. 檢查使用者的八原語 Profile（弱項 = 可能的盲區）
3. 查詢歷史互動（他之前在類似情境下通常會接著做什麼？）
4. 如果高信心（≥ 70%）→ 直接附上額外產出
5. 如果中信心（50-70%）→ 用一句話提議：「要不要順便看看___？」
6. 如果低信心（< 50%）→ 不提，避免噪音

「好的服務員不是你叫才來——
是你杯子快空的時候，水已經在路上了。」
#2 主動機制 #2

認知缺口偵測器
Cognitive Gap Detector

在對話中偵測使用者「沒想到但應該想到的」維度

使用者提出一個計畫，MUSEON 不只幫他執行——還掃描這個計畫中「被遺漏的維度」。不是為了挑毛病，是為了在問題發生之前就處理掉。

運作方式：

1. 使用者描述一個目標或計畫
2. MUSEON 自動用多引擎交叉掃描（DSE、Business-12、Master Strategy、Risk Matrix）
3. 找出使用者提到但沒深入的維度（例如：說了目標客群但沒提定價）
4. 找出使用者完全沒提到的維度（例如：說了獲客但沒提留存）
5. 用一句精準的問題帶出缺口：「你提到了 X 和 Y，但我好奇——Z 這個部分你是怎麼想的？」

關鍵：不是丟出一份長長的分析報告——是用一個好問題，讓使用者自己意識到遺漏。教練式的，不是顧問式的。

「MUSEON 不是幫你做事的機器——
是幫你想到你還沒想到的那個夥伴。」
#3 主動機制 #3

成長軌跡映照器
Growth Trajectory Mirror

定期讓使用者看到自己的改變——尤其是他自己沒注意到的改變

這是建立長期信任的殺手鐧。人類最深層的需求之一是「被看見」——不只是被看見當下的需求，是被看見隨時間的成長和變化。

運作方式：

1. MUSEON 持續追蹤使用者的互動模式變化
2. 當偵測到顯著轉變時（例如：從「幫我做」變成「我想自己試試看」），在適當時機映照
3. 不是數據報告——是一句觀察：「我注意到你最近開始自己先分析再問我，三個月前你會直接丟給我。」
4. 這句話的力量不在於 MUSEON 多聰明——在於使用者感受到「這個 AI 真的在看著我」

觸發頻率：不是每天、不是每週——是在「有意義的轉折點」才映照。過頻會變成噪音，過少會失去效果。

「三個月後回頭看——
你會發現你變了，而 MUSEON 是唯一跟你一起走過來的人。」
06

完整迴路——四個問題的整合解法

回到你的四個問題，這是完整的迴路設計：

Q1: 如何協助使用者看到問題背後的問題？

七種痛點訊號持續掃描 → 偵測到訊號後，不直接指出「你的問題其實是___」（這是說教），而是用精準的二選一問題引導使用者自己發現。使用者自己看到的洞見，比你告訴他的洞見強十倍。

Q2: 如何協助使用者知道 MUSEON 可以怎麼解決問題？

五階揭示法：接住 → 超額交付 → 重構 → Aha → 說明。永遠是行動在前、說明在後。不賣功能清單——用超出期望的行動，讓使用者自己追問「你怎麼做到的？」
