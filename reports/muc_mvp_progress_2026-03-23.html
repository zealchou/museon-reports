系統範圍（白話版）

MUC MVP 一句話定義：讓客戶能付錢、拿到授權、在設備上合法使用。整個系統只做三件事 — 收錢 → 發授權 → 管設備。

12 個核心模組

🔑
帳號系統
註冊、登入 = 辦會員卡
💳
付款資料
客戶的發票資訊 = 留下匯款帳號
🎫
序號管理
一把鑰匙一組序號 = 軟體授權碼
📋
方案管理
訂閱 vs 續約、不同價格 = 菜單
🔄
訂閱週期
付了→啟用→到期→寬限→停權 = 健身房月費
💰
人工收款
目前手動對帳確認 = 先用匯款
📱
設備配對
Agent 跟伺服器握手 = 手機配對藍牙
🛡
設備授權
定期驗證「你還能用嗎」= 門禁卡刷一下
✅
接單前檢查
每次工作前確認授權 = 出勤打卡
🔃
換機/停用
換新設備要重新配對 = SIM 卡換手機
🚨
告警通知
異常、到期、收款結果通知 = 保全警報
📝
稽核紀錄
所有重要操作留痕跡 = 監視器錄影

安全基線（已完成）

  • Ed25519 數位簽章驗證 — Agent 內建公鑰，每次授權回應都驗簽章
  • 配對碼一次性、短時效（5 分鐘）、只存 hash 不存明文
  • Device Token 高熵亂數、只存 hash、可隨時撤銷
  • 防重放攻擊 — nonce + timestamp 雙重驗證
  • 離線超過 15 分鐘自動停止接新任務
  • 完整稽核紀錄（append-only，不可刪除）

商業判斷重點

現在就能收錢。 MVP 12 個模組全部完成，金流用人工對帳（匯款 + 人工確認），不用等金流串好。完整的自助流程已經可以 demo：註冊 → 選方案 → 付款 → 拿序號 → 配對設備 → 開始使用 → 點數消耗可視。
規模化瓶頸在 Phase 15。 超過 10 個客戶就會手動對帳累死。金流商決策是最緊急的決定 — 決定了才能動工。
